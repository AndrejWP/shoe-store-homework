# 👟 BLACK STORE — Интернет-магазин обуви

Учебный проект по дисциплине «Конструирование и сопровождение программ» (КСП).

## О проекте

BLACK STORE — это фронтенд интернет-магазина обуви с каталогом товаров, фильтрацией по категориям и корзиной покупок. Корзина сохраняется в `LocalStorage`, поэтому товары не пропадут после перезагрузки страницы.

## Функционал

- Каталог товаров с карточками (название, фото, описание, цена)
- Фильтрация по категориям: кроссовки, ботинки, классическая обувь
- Корзина: добавление, удаление товаров, подсчёт итоговой суммы
- Сохранение корзины в `LocalStorage`
- Оплата и очистка корзины
- Авторизация на PHP: вход, выход, сессии и логирование
- REST API на PHP для работы с пользователями

## Стек

- HTML5
- CSS3
- JavaScript (Vanilla)
- PHP 8.x

## Запуск

Открыть `index.html` в браузере. Сборка не требуется.

Для проверки PHP-авторизации проект нужно запускать через локальный веб-сервер
XAMPP / OpenServer / Denwer или аналог.

Тестовые пользователи:

- `admin` / `password`
- `user` / `password`

Логи авторизации записываются в файл `logs/auth.log`.

## REST API для практической 6

API запускается через локальный веб-сервер XAMPP / OpenServer / Denwer.

Основные endpoint'ы:

- `POST /api/v1/register` - регистрация пользователя
- `POST /api/v1/login` - авторизация пользователя
- `GET /api/v1/users` - список пользователей
- `GET /api/v1/users/1` - один пользователь по id
- `PUT /api/v1/users/1` - изменение пароля пользователя
- `PATCH /api/v1/users/1` - изменение пароля пользователя
- `DELETE /api/v1/users/1` - удаление пользователя

Если на сервере не работает `.htaccess`, можно использовать вариант через `index.php`:

- `/api/index.php/v1/register`
- `/api/index.php/v1/users`

Пример JSON для регистрации:

```json
{
  "name": "Nikita",
  "email": "nikita@test.com",
  "password": "123456"
}
```

Пользователи сохраняются в файл `data/users.json`.

Основной сценарий работы:

1. Пользователь открывает `login.php`.
2. Форма отправляет логин и пароль методом `POST` в `process_login.php`.
3. PHP ищет пользователя в массиве `users.php`.
4. Пароль проверяется через `password_verify`.
5. При успехе создаётся сессия `$_SESSION`.
6. Пользователь попадает в `cabinet.php`.
7. При выходе открывается `logout.php`, сессия уничтожается.
