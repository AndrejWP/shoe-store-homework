# Практическая работа 5

## Тема

Применение PHP. Авторизация и логирование.

## Цель работы

Получить навыки практической работы с PHP при создании формы авторизации,
проверке логина и пароля, работе с сессиями и записи логов авторизации в файл.

## Используемые инструменты

- Visual Studio Code
- Локальный веб-сервер XAMPP / OpenServer / Denwer
- PHP 8.x
- Браузер

## Что было реализовано

В проект интернет-магазина BLACK STORE была добавлена простая авторизация на PHP.

Реализованы:

- форма входа пользователя;
- проверка логина и пароля методом POST;
- хранение пользователей в PHP-массиве;
- проверка пароля через `password_verify`;
- создание сессии при успешном входе;
- закрытая страница личного кабинета;
- выход из аккаунта через уничтожение сессии;
- запись событий авторизации в файл `logs/auth.log`.

## Структура добавленных файлов

- `login.php` - форма авторизации.
- `process_login.php` - обработка формы и проверка пользователя.
- `users.php` - массив пользователей и хэши паролей.
- `cabinet.php` - закрытая страница, доступная только после входа.
- `logout.php` - выход из системы.
- `logger.php` - функция записи логов.
- `logs/auth.log` - файл логов, создаётся автоматически.

## Тестовые данные

Для проверки используются два пользователя:

- логин: `admin`, пароль: `password`;
- логин: `user`, пароль: `password`.

Пароли в массиве пользователей хранятся не открытым текстом, а в виде хэша.

## Описание работы программы

1. Пользователь открывает страницу `login.php`.
2. Вводит логин и пароль.
3. Форма отправляет данные методом POST в файл `process_login.php`.
4. PHP проверяет, существует ли пользователь в массиве `users.php`.
5. Если пользователь найден, введённый пароль проверяется функцией `password_verify`.
6. При успешной проверке создаётся сессия:

```php
$_SESSION['user_id'] = $user['id'];
$_SESSION['login'] = $login;
```

7. После входа пользователь перенаправляется в `cabinet.php`.
8. Если пользователь не вошёл, доступ к `cabinet.php` запрещён.
9. При выходе открывается `logout.php`, где сессия очищается и уничтожается.

## Логирование

Все попытки входа и выхода записываются в файл `logs/auth.log`.

В лог попадают:

- дата и время события;
- IP-адрес пользователя;
- логин;
- действие;
- дополнительная информация.

Пример строки лога:

```text
2026-04-30 12:10:05 | ip=127.0.0.1 | login=admin | action=SUCCESS_LOGIN
```

Возможные действия:

- `SUCCESS_LOGIN` - успешный вход;
- `FAIL_LOGIN` - неудачный вход;
- `LOGOUT` - выход из системы.
